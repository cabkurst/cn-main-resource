# cn-main-resource

一个用于归档和发布独立 HTML 页面的资源仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的存放与分发集合。

## 项目简介

本项目旨在收集、整理并托管多个独立的 HTML 页面。这些页面可能用于展示信息、提供工具或作为其他用途的静态资源。仓库以通用性为原则，不绑定具体业务或站点。

## 目录说明

```
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── index.html      # 仓库首页/导航页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名清晰，便于识别。
- **assets/**：按类型或页面组织资源文件，避免路径混乱。
- **index.html**：如果存在，可作为仓库的入口页面，列出所有已归档页面的链接。

## 页面归档说明

- 所有页面均为独立完整的 HTML 文件，可直接通过浏览器打开或部署到静态托管服务。
- 页面内容不涉及任何推广、营销或诱导性信息，仅提供原始功能或展示。
- 归档时请保持页面结构清晰，避免外部依赖（如 CDN 链接）过多，以提高离线可用性。

## 维护说明

- 欢迎提交新增页面或改进现有页面，请遵循以下原则：
  - 页面应自包含，尽量不依赖外部资源。
  - 新增页面请放入 `pages/` 目录，并更新 `index.html`（如存在）。
  - 保持目录结构整洁，避免重复或冗余文件。
- 本仓库仅作为静态资源存储，不提供后台服务或动态功能。
- 如发现页面内容存在错误或失效，请提交 Issue 或直接发起 Pull Request。
